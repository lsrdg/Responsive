Plugins em JavaScript do Responsive

O Responsive vem com uma série de plugins de jQuery e métodos que provêm componentes comuns em páginas da rede, em um formato adaptável.

Confira os detalhes em cada seção.

Fortalecido com jQuery

Todos os plugins dependem da jQuery (isso significa que a jQuery deve ser incluída antes dos arquivos do plugin). Consulte bower.json para ver quais versões de jQuery são suportadas.

Cada plugin também foi construído de forma a ser extensível, provendo eventos que podem ser aproveitados para que você melhore ainda mais a sua página.

Atenção!

Dentro núcleo de JavaScript do Responsive nós temos uma função html() de jQuery estendida para disparar o evento: domchanged. Isso permite ao desenvolvedor conectar e manusear o evento e responder ao conteúdo de forma dinâmica ao usar esse método.

Acessibilidade

Todos os plugins do Responsive foram construídos desde a base para serem acessíveis com pelo mouse, por toque e pelo teclado. Eles também adicionam a atribuição ARIA apropriada quando necessário, tudo para garantir que leitores de tela possam acompanhar a funcionalidade de cada plugin.

Data-API - Dados da Interface de Programação de Aplicação

É possível rodar os plugins sem escrever uma linha sequer de JavaScript, pois todos os plugins vêm com uma marcação para a interface de programação de aplicação (API) voltada para a atribuição de dados. Essa é API do Responsive de primeira classe, e deveria ser a primeira a ser considerada quando usando os plugins. Essa API de uma forma similar, porém melhora em relação ao jeito que a marcação da API do Bootstrap funciona.

Para permitir situação em que você queira desativar a funcão, é provido um método que permite a habilidade de desativar a API, desconectando todos os os eventos no documento nomeados com data-api. 

// Desativar globalmente a API de dados do Responsive.
$(document).off(".data-api");

Cada plugin pode ser individualmente desativado, bastando prefixar o nome do plugin ao nome.

// Desativar a API de dados do Responsive para o plugin de Carrossel.
$(document).off(".r.carousel.data-api");

When plugins are added to the page using this API they will also automatically register when loaded dynamically. function.

API Programática

Assim como a API de dados recomendada, a Responsive também vem com uma API programática que pode ser chamada pelo JavaScript. Cada plugin roda com o formato jQuery padrão com uma única estrutura interligável. 

// Crie uma janela e um elemento de alvo modal com a id 'modal-target'
// e mostre-a imediatamente.
$("<a/>").modal({target: "#modal-target", immediate : true});

Colisão de nomenclaturas

Muitos plugins jQuery dentro de bibliotecas para a interface do usuário ('UI frameworks') usam as mesmas convenções para os nomes, e para evitar colisões cada plugin recebeu a habilidade de reatribuir sua própria funcionalidade para um novo objeto por meio da chamada de método .noConflict() method call. Isso permite que dois plugins conflitantes coexistam lado a lado. 

// Reatribuir o plugin de Carrossel adaptável um novo nome de função. 
var responsiveCarousel = $.fn.carousel.noConflict();
$.fn.responsiveCarousel = responsiveCarousel;
