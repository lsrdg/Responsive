Navegação

Um plugin jQuery que adiciona uma navegação fora da tela à página. Faz uso de transições de CSS para executar a animação.

Vem com uma API de marcação que permite rodar o plugin sem precisar escrever uma linha sequer de JavaScript. Essa é a API de primeira linha do Responsive e deveria ser a primeira a ser considerada ao fazer uso de algum plugin.

Se nenhum botão estiver presente como o primeiro descendente da navegação, o plugin irá automaticamente adicionar um como um gatilho.
Para renderizar do lado oposto da página, adiciona a classe .pull à navegação.

Abrir a navegação tranca a página para prevenir que haja barras de rolagem duplicadas em certas janelas de exibição.

Accessibility

On initialisation a clone of the navigation landmarks is placed, hidden at the bottom of the page to ensure that the landmarks from the landmarks list dialog are registered and accessible. The appropriate roles and aria attributes are also added to the button and navigation elements.

When open pressing tab or will focus elements within the menu only shift + tab Will allow you to exit from the top. Pressing esc will close the menu.

Right-to-Left Language Support

Adding the attribute dir="rtl" to the html tag will reverse the position of the navigation.

Examples

This website uses the navigation plugin as its main navigation.

Markup

<nav data-navigation> 
    <button>Menu</button> 
    <ul><li><a href="LINK_1_URL">Link 1</a></li></ul> 
</nav>

Methods

The navigation plugin exposes the following method signatures.

.navigation()
Initialises the plugin.
.navigation("show")
Show the navigation.
.navigation("hide")
Hide the navigation.

Events

The navigation plugin exposes the following events allowing the developer to tap into its behaviour.

show.r.navigation
This event is fired immediately when the plugins show instance method is invoked.
shown.r.navigation
This event is fired when the plugins shown instance method is invoked once the target has completed the animation and is shown.
hide.r.navigation
This event is fired immediately when the plugins hide instance method is invoked.
hidden.r.navigation
This event is fired when the plugins hidden instance method is invoked once the target has completed the animation and is hidden.

Binding to these events is as follows:

// Bind to the show event.
$("YOUR_SELECTOR").on("show.r.navigation", function(event) {
    // Your custom behaviour...
});
                                               
// Bind to the shown event.
$("YOUR_SELECTOR").on("shown.r.navigation", function(event) {
    // Your custom behaviour...
});
// Bind to the hide event.
$("YOUR_SELECTOR").on("hide.r.navigation", function(event) {
    // Your custom behaviour...
});
// Bind to the hidden event.
$("YOUR_SELECTOR").on("hidden.r.navigation", function(event) {
    // Your custom behaviour...
});

Data API

The navigation plugin's default behaviour is bound using a markup API that allows running the plugin without writing a single line of JavaScript. If you need to override this behaviour the following code will allow you to do so. 

// Disable the default loading behaviour
$(document).off(".r.navigation.data-api");
