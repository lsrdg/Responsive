Condicional

Um plugin jQuery que substitui o conteúdo de dentro de uma página baseado no nível da grelha (xxs, xs, s, m, l) em que a janela de exibição atual se enquadra.

O plugin usa ajax para carregar trazer o conteúdo definido. Os caminhos de url para o conteúdo são definidos no mesmo formato do método Load() do jQuery, que permite que você especifíque seletores. Após cada chamada inicial, o conteúdo é guardado em cache, de modo que nenhum pedido adicional é feito. Se nada for definido, como alternativa, qualquer html contido dentro do elemento do plugin será salvo e guardado em cache.

Vêm com uma API de marcação que permite rodar o plugin sem precisar escrever uma linha sequer de JavaScript. Essa é a API de primeira classe do Responsive e deveria ser a primeira a ser considerada ao usar um plugin.

Redimensione a janela do navegador para ver a mudança dinâmica do conteúdo carregado.

Métodos

O plugin Conditional apresenta os seguintes métodos de assinatura.

.conditional(options)
Inicializa o plugin com um objeto opcional de opções (object).
options
Contém parâmetros para passar para o plugin.
xxs
O caminho para o conteúdo a ser carregado quando a grelha está no modo xxs.
xs
O caminho para o conteúdo a ser carregado quando a grelha está no modo xs.
s
The path to the content to load when the grid is in s mode.
m
The path to the content to load when the grid is in m mode.
l
The path to the content to load when the grid is in l mode.
fallback
The path to the content to load when no specific content to load has been defined for each mode.
.conditional("resize")
Triggers the plugin to reload the content.

Events

The conditional plugin exposes the following events allowing the developer to tap into its behaviour.

load.r.conditional
This event is fired when the plugins resize instance method is invoked and content is about to be loaded.
loaded.r.conditional
This event is fired when the plugins resize instance method is invoked and content is loaded. The event data contains three custom properties:
relatedTarget
The containing element that the plugin is bound to.
loadtarget
The url to the target html to load.
grid
The current grid range in which the event is firing. xs, s, m, or l
error.r.conditional
This event is fired when the plugin encounters an error when attempting to load ajax content. The event data contains three custom properties:
relatedTarget
The containing element that the plugin is bound to.
loadtarget
The url to the target html to load.
grid
The current grid range in which the event is firing. xs, s, m, or l

Binding to these events is as follows:

// Bind to the load event.
$("YOUR_SELECTOR").on("load.r.conditional", function(event) {
    // Your custom behaviour...
});

// Bind to the loaded event.
$("YOUR_SELECTOR").on("loaded.r.conditional", function(event) {
    // Your custom behaviour...
});
                                               
// Bind to the error event.
$("YOUR_SELECTOR").on("error.r.conditional", function(event) {
    // Your custom behaviour...
});

Data API

The conditional's default behaviour is bound using a markup API that allows running the plugin without writing a single line of JavaScript. If you need to disable this behaviour the following code will allow you to do so. 

// Disable the default loading behaviour
$(document).off(".r.conditional.data-api");
