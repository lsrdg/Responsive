Condicional

Um plugin jQuery que substitui o conteúdo de dentro de uma página baseado no nível da grelha (xxs, xs, s, m, l) em que a janela de exibição atual se enquadra.

O plugin usa ajax para carregar trazer o conteúdo definido. Os caminhos de url para o conteúdo são definidos no mesmo formato do método Load() do jQuery, que permite que você especifíque seletores. Após cada chamada inicial, o conteúdo é guardado em cache, de modo que nenhum pedido adicional é feito. Se nada for definido, como alternativa, qualquer html contido dentro do elemento do plugin será salvo e guardado em cache.

Vêm com uma API de marcação que permite rodar o plugin sem precisar escrever uma linha sequer de JavaScript. Essa é a API de primeira classe do Responsive e deveria ser a primeira a ser considerada ao usar um plugin.

Redimensione a janela do navegador para ver a mudança dinâmica do conteúdo carregado.

Métodos

O plugin Conditional apresenta os seguintes métodos de assinatura.

.conditional(options)
Inicializa o plugin com um objeto opcional de opções (object).
options
Contém parâmetros para passar para o plugin.
xxs
O caminho para o conteúdo a ser carregado quando a grelha está no modo xxs.
xs
O caminho para o conteúdo a ser carregado quando a grelha está no modo xs.
s
O caminho para o conteúdo a ser carregado quando a grelha está no modo s.
m
O caminho para o conteúdo a ser carregado quando a grelha está no modo m.
l
O caminho para o conteúdo a ser carregado quando a grelha está no modo l.
fallback
O caminho para o conteúdo a ser carregado quando nenhum conteúdo for especificado para carregar para cada modo.
.conditional("resize")
Ativa o plugin para recarregar o conteúdo.

Eventos

O plugin Conditional apresenta os seguintes eventos para permitir ao desenvolvedor manipular o seu comportamento.

load.r.conditional
Este evento é ativado quando a instância resize do método é invocada e o conteúdo está prestes a ser carregado.
loaded.r.conditional
Este evento é ativado quando a instância resize do método é invocada e o conteúdo está carregado. O evento de dado contém três propriedades personalizadas:
relatedTarget
O elemento que contém o plugin, e ao qual o plugin está limitado.
loadtarget
O url ao alvo de html a ser carregado.
grid
O alcance da grelha atual, na qual o evento está sendo ativado. xs, s, m, or l
error.r.conditional
Este evento é ativado quando o plugin encontra um erro ao tentar carregar conteúdo em ajax. O evento de dado contém três propriedades personalizadas:
relatedTarget
O elemento que contém o plugin, e ao qual o plugin está limitado.
loadtarget
O url ao alvo de html a ser carregado.
grid
O alcance da grelha atual, na qual o evento está sendo ativado. xs, s, m, or l

A conexão a esses eventos é feita da seguinte maneira:

// Conectar ao evento 'load'. 
	$("SEU_SELETOR").on("load.r.conditional", function(event) {
    // Seu comportamento personalizado...
});

// Conectar ao evento 'loaded'.
$("SEU_SELETOR").on("loaded.r.conditional", function(event) {
    // Seu comportamento personalizado...
});
                                               
// Conectar ao evento 'error'.
$("SEU_SELETOR").on("error.r.conditional", function(event) {
    // Seu comportamento personalizado...
});

API de dados

O comportamento padrão do plugin Conditional é limitado pelo uso da API de marcação que permite rodar o plugin sem precisar escrever uma linha sequer de JavaScript. O seguinte código irá lhe auxiliar caso precise desativar esse comportamento. 

// Desativar o comportamento de carregamento padrão
$(document).off(".r.conditional.data-api");
