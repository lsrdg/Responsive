Carrossel

Um plugin que cria um plugin com jQuery que cria um carrossel para circular através de elementos. Usa transições de CSS para executar a animação.

O carrossel vem em dois modos: com deslizamento ("slides") ou com desvanecimento ("fade"), ambos animados por meio de transições de CSS3. Em navegadores que não suportam transições, eles irão simplesmente mudar sem nenhum efeito especial.

É possível rodar os plugins sem escrever uma linha sequer de JavaScript, pois todos os plugins vêm com uma marcação para a interface de programação de aplicação (API) voltada para a atribuição de dados. Essa é API do Responsive de primeira classe, e deveria ser a primeira a ser considerada quando usando os plugins. Comes with a markup API that allows running the plugin without writing a single line of JavaScript. This is Responsive's first class API and should be your first consideration when using a plugin.

Acessibilidade

O carrossel, por padrão, não fazer transições automaticamente. Isso faz com que leitores de tela tenham menos problemas ao acompanhar a página.

Em dispositivos que suportam toque, o carrossel irá iterar através da coleção de itens arrastando para a esquerda ou para a direita. Também ser utilizado o suporte ao teclado para navegar, quando o carrossel estiver em foco, usando as teclas para a ESQUERDA e para a DIREITA.

Suporte a escritas da direita para a esquerda

Adicionando o atributo dir="rtl" a tag html irá inverter a ordem dos itens do carrossel e todos os outros comportamentos de inserção serão invertidos.

Métodos

O Carousel expões as seguintes assinaturas de métodos ("method signatures").

.carousel(opções)
Inicia o plugin com um object de opção opcional.
Nome Tipo Descrição Valor padrão
mode cadeia de caracteres ("string") A cadeia de caracteres containing either the values fade or slide. slide
interval número The time in milliseconds between carousel transitions. 0 (Better for accessibility)
pause cadeia de caracteres The event to bind the pause behaviour to hover
wrap booleano Whether the carousel items should wrap true
keyboard booleano Whether the carousel responds to keyboard events true
touch booleano Whether the carousel responds to touch events true
lazyLoadImages booleano Whether to lazily load any images on page load. true
lazyOnDemand booleano Whether to lazily load any images on demand when sliding. Overwrites lazyLoadImages true
nextTrigger cadeia de caracteres A jQuery selector to define any external element to trigger the next() method. null
nextHint cadeia de caracteres A hidden html value that is added to the next trigger for accessibility purposes. The default language direction indicator is reversed by setting the dir="rtl" attribute on the html element. Next {Left|Right} Arrow
previousTrigger cadeia de caracteres A jQuery selector to define any external element to trigger the prev() method. null
previousHint cadeia de caracteres A hidden html value that is added to the previous trigger for accessibility purposes. The default language direction indicator is reversed by setting the dir="rtl" attribute on the html element. Previous {Left|Right} Arrow
indicators cadeia de caracteres A jQuery selector to define any external indicators <ol/> that highlight the carousels current position null
.carousel("cycle")
Cycles through the carousel instance items in a logical order.
.carousel("pause")
Triggers the pause event of the carousel instance stopping it cycling.
.carousel("prev")
Cycles the carousel instance to the previous item.
.carousel("next")
Cycles the carousel instance to the next item.
.carousel(number)
Cycles the carousel to a particular item with at the given index (zero based).

Events

The carousel exposes the following events allowing the developer to tap into its behaviour.

slide.r.carousel
This event is fired immediately when the plugins slide instance method is invoked. The event data contains two custom properties:
relatedTarget
The carousel item that will be next in the logical order.
direction
The direction in which the carousel is transitioning.
slid.r.carousel
This event is fired when the plugins slid instance method is invoked once the carousel has completed sliding. The event data contains two custom properties:
relatedTarget
The carousel item that will be next in the logical order.
direction
The direction in which the carousel is transitioning.

Binding to these events is as follows:

// Bind to the slide event.
$("YOUR_SELECTOR").on("slide.r.carousel", function(event) {
    // Your custom behaviour...
});
                                               
// Bind to the slid event.
$("YOUR_SELECTOR").on("slid.r.carousel", function(event) {
    // Your custom behaviour...
});

Data API

The carousel's default behaviour is bound using a markup API that allows running the plugin without writing a single line of JavaScript. If you need to disable this behaviour the following code will allow you to do so. 

// Disable the default loading behaviour
$(document).off(".r.carousel.data-api");
