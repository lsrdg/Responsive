Elementos de formulários

Formulários que funcionam em múltiplos navegadores, que são normalizados, adaptáveis, e com estilo minimalista, tudo para que você possa sobreescrever e obter um CSS mais limpo.

O Responsive não requer que você adicione classes extras só para estilizar campos de inserção de dados em formulários. Pelo contrário, todos os campos válidos de inserção de dados em HTML5 foram estilizados para serem adaptáveis, com melhorias progressivamente implementadas para tipos não suportados.

Tipos de HTML5 suportados: text, password, email, url, search, tel, datetime, datetime-local, date, month, time, week, number, range, e color.

A classe .readonly pode ser usada como gancho para simular campos não-editáveis, e todos os campos podem ser configurados para aparecer na linha, precisando apenas adicionar a classe .inline-block.

Suporte a línguas com escrita da direita para a esquerda

Adicionar o atributo dir="rtl" para a tag html irá inverter o layout de campos para inserção de dados para permitir escritas da direita para a esquerda. of form inputs to allow for right-to-left languages.

Formulários verticais

Os elementos de formulários do Responsive abrangem 100% por padrão; a largura deles foi desenhada para ser restringida pela grade que existe no sistema. Isso faz deles elementos extremamente flexíveis.

Entrada de dados agrupada (Grouped inputs)

Dados podem ser agrupados com prefixos e sufixos anexados, bastando usar um container com a classe .input-group para agrupá-los. Esse grupos são inteiramente responsáveis e irão se adaptar a largura total do container que os contém.

Adicionar a classe .input-group-addon .supplement aos vãos (spans) dentro do grupo de inserção (input group) irá estilizá-los apropriadamente.

IOS

Em dispositivos IOS, a cor (chroma) é retirada da seleção e de certos campos de seleção, permitindo que eles sejam estilizados com a mesma flexibilidade que navegadores em computadores (desktops). Como tal, o indicador de menu suspenso (dropdown menu) precisa ser substituído. Isso é feito usando uma imagem svg com dado codificados, embutida no CSS.
