O Responsive é a biblioteca dos desenvolvedores.

Acessível

A acessibilidade tem prioridade dentro do Responsive e como tal, cada componente e plugin dentro da biblioteca foi cuidadosamente criado para ser totalmente acessível, tanto para teclados quanto para leitores de tela.

Flexível

O Responsive possui suporte nativo para línguas de escrita da direita para a esquerda. O suporte é para todos os componentes, até mesmo eventos de toque foram alterados para prover a melhor experiência possível para usuários de línguas da direita para a esquerda.

Extensível

Construído com Sass e jQuery, as melhores ferramentas em suas respectivas áreas, o Responsive foi contruído para cobre todas as necessidades de uma página moderna. Com isso dito, a biblioteca também pode ser estendida e aprimorada para se adequar as suas necessidades específicas.

Por quê Responsive?

Outras bibliotecas dão muita opinião em como que deve ser o desenho da página, além de terem estilos pesados. São ótimas para criar protótipos, mas cada vez que você encara de frente um projeto real com elas, você acaba tendo que sobreescrever muitos estilos de desenhos que não fazem nada para adicionar as funcionalidades da página. E isso custa tempo e recursos para desenvolvedores.

A Responsive tem sido contruída com isso em mente. Ela é o resultado de de milhares de horas de trabalho real, desenvolvimento de rede conduzido pelo/para a cliente e com testes; especificamente desenhada para ser tão leve quanto possível para prevenir a necessidade de desfazer estilos impostos pela biblioteca em si e permitir que desenvolvedoras escrevam código eficiente e de baixo custo.

A Responsive é minúscula. A saída combinada de CSS e JavaScript é de apenas {{site.buildsize}}kb minificados e zipados em formato gzip, porém existe um bocado de funcionalidades construídas na biblioteca como suporte ao toque, línguas da direita para a esquerda, e acessibilidade. Foi desenhada para ser jogada, assim como está, em sua página, da mesma forma que você faria com a Normalize.css.
